# API Key配置和真实测试指南

## ⚠️ 重要提示

本指南将指导你完成真实的API配置和发布测试。请注意：

1. **需要真实的OpenAI API Key** (或兼容服务的Key)
2. **需要小红书账号** 用于登录和发布
3. **会产生实际费用** (API调用费用)
4. **建议先使用Dry-Run模式** 验证流程

---

## 📋 准备工作

### 1. 获取API Key

#### 选项A: OpenAI官方 (推荐用于生产)

1. 访问: https://platform.openai.com/api-keys
2. 登录OpenAI账号
3. 创建新的API Key
4. 复制Key (格式: `sk-...`)

**预计费用** (单篇):
- 文案生成 (GPT-4o): ~$0.02
- 图片生成 (DALL-E 3, 3张): ~$0.36
- **总计**: ~$0.38/篇

#### 选项B: DeepSeek (推荐用于测试)

1. 访问: https://platform.deepseek.com/
2. 注册并登录
3. 创建API Key
4. 复制Key

**优势**:
- 更便宜 (约OpenAI的1/10)
- 中文支持更好
- 速度更快

**注意**: DeepSeek不支持图片生成，需要使用本地图片或配合OpenAI

### 2. 准备小红书账号

- 有效的小红书账号
- 能够登录创作者中心
- 建议使用测试账号

## 🔧 配置步骤

### 方法1: 使用OpenAI (完整功能)

```bash
# 1. 编辑配置文件

# 2. 修改以下内容

OPENAI_BASE_URL=https://api.openai.com/v1
```

如果只想测试优化功能，可以使用模拟脚本（不调用真实API）。

### 阶段2: 测试API连接 (小费用 ~$0.02)

创建测试脚本验证API是否可用：

```bash
# 创建测试脚本
cat > test_api_connection.py << 'EOF'
"""测试API连接"""
from openai import OpenAI
from config import OPENAI_API_KEY, OPENAI_BASE_URL, CHAT_MODEL

print("🔍 测试API连接...")
print(f"Base URL: {OPENAI_BASE_URL}")
print(f"Model: {CHAT_MODEL}")

try:
    client = OpenAI(api_key=OPENAI_API_KEY, base_url=OPENAI_BASE_URL)
    
    # 测试简单调用
    response = client.chat.completions.create(
        model=CHAT_MODEL,
        messages=[{"role": "user", "content": "你好，测试连接"}],
        max_tokens=10
    )
    
    print(f"✅ API连接成功！")
    print(f"响应: {response.choices[0].message.content}")
    
except Exception as e:
    print(f"❌ API连接失败: {e}")
    print("\n💡 请检查:")
    print("  1. API Key是否正确")
    print("  2. Base URL是否正确")
    print("  3. 账户余额是否充足")
EOF

# 运行测试
python test_api_connection.py
```

## 🐛 常见问题处理

### 问题1: API Key无效

**症状**:
```
❌ 文案生成失败: Invalid API key
```

**解决**:
1. 检查`.env`文件中的Key是否正确
2. 确认Key没有多余空格
3. 确认Key格式正确 (sk-...)
4. 检查账户余额

### 问题2: API限流

**症状**:
```
⚠️ 第 1 次尝试失败: 429 Too Many Requests
⚠️ 第 2 次尝试失败: 429 Too Many Requests
✅ 第 3 次尝试成功
```

**说明**: 
- 这是正常的，重试机制会自动处理
- 如果频繁出现，说明请求过快
- 可以增加间隔时间或减少并发数

### 问题3: Cookie失效

**症状**:
```
❌ Cookie 已失效，请重新登录
```

**解决**:
```bash
python main.py login
```

### 问题4: 图片生成失败

**症状**:
```
❌ 图片生成失败: Content policy violation
```

**解决**:
1. 修改主题，避免敏感内容
2. 或使用本地图片：`--images ./test_images/`
