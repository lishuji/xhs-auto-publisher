# 实际测试指南

本指南帮助你完成小红书自动发布工具的实际测试验证。

## 📋 测试前准备

### 1. 安装依赖

```bash
cd /Users/kanelli/workspace/xhs-auto-publisher

# 安装 Python 依赖
pip3 install -r requirements.txt

# 安装 Playwright 浏览器（仅首次需要）
playwright install chromium
```

预计安装时间：2-3分钟

### 2. 配置环境变量

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置文件
vim .env
# 或者
nano .env
```

**必填项**：
```env
OPENAI_API_KEY=sk-your-actual-api-key-here
```

**可选优化配置**：
```env
# 使用 DeepSeek（更快更便宜）
OPENAI_BASE_URL=https://api.deepseek.com/v1
CHAT_MODEL=deepseek-chat
```

### 3. 验证配置

```bash
# 检查依赖
python3 -c "import openai; import playwright; print('✅ 依赖正常')"

# 检查配置文件
python3 -c "from config import OPENAI_API_KEY; print('✅ 配置正常' if OPENAI_API_KEY != 'sk-your-api-key-here' else '⚠️ 请配置API Key')"
```

## 🧪 测试流程

### 阶段1: 首次登录（约2分钟）

```bash
python3 main.py login
```

**预期行为**：
1. 自动打开 Chromium 浏览器
2. 跳转到小红书创作者中心登录页
3. 使用小红书APP扫码或手机号登录
4. 登录成功后自动保存Cookie

**成功标志**：
```
✅ 登录成功
💾 Cookie 已保存到: cookies/xhs_cookies.json
```

---

### 阶段2: 单篇发布测试（约1分钟）

#### 测试2.1: Dry-Run模式（不真正发布）

```bash
python3 main.py publish --topic "测试主题：春天的美食" --dry-run
```

**测试目的**：
- ✅ 验证文案生成（含重试机制）
- ✅ 验证图片并发生成
- ✅ 验证浏览器自动化
- ❌ 不真正点击发布按钮

**预期耗时**：
- 文案生成: 5-10秒
- 图片生成: 30-40秒（并发优化）
- 浏览器填充: 10-15秒
- **总计: 约60秒**

**成功标志**：
```
📝 Step 1: 生成文案 - 主题: 测试主题：春天的美食
✅ 文案生成成功: [标题]
🎨 Step 2: 准备图片
🚀 并发生成 3 张图片...
✅ 图片 1 完成
✅ 图片 2 完成
✅ 图片 3 完成
📸 共生成 3/3 张图片
🚀 Step 3: 发布到小红书
🧪 Dry-Run 模式：填充内容但不发布
🎉 笔记 [标题] 发布流程完成！
```

#### 测试2.2: 观察重试机制

如果遇到网络波动，应该看到：
```
⚠️ 第 1 次尝试失败: Connection timeout, 将在 2.0 秒后重试...
✅ 第 2 次尝试成功
```

#### 测试2.3: 真实发布（可选）

```bash
python3 main.py publish --topic "实际测试主题" --extra "注明这是测试笔记"
```

⚠️ **注意**：这会真正发布到小红书，建议先用Dry-Run模式验证。

### 阶段3: 批量发布测试（约10-15分钟）

#### 测试3.1: 创建测试主题列表

```bash
cat > test_topics.txt <<EOF
春天的第一杯奶茶
周末居家办公小技巧
简单又好吃的快手菜
EOF
```

#### 测试3.2: 批量Dry-Run（优化模式）

```bash
python3 main.py batch --file test_topics.txt --dry-run
```

**测试目的**：
- ✅ 验证批量发布优化（智能预生成）
- ✅ 验证间隔控制
- ✅ 验证失败容错

**预期行为**：
```
📋 共 3 个主题待发布
🚀 启用优化模式：发布间隔期预生成下一篇内容

📌 发布第 1/3 篇: 春天的第一杯奶茶
🔄 后台生成: 周末居家办公小技巧  ← 预生成开始
⏰ 等待 300 秒，同时生成下一篇...
✅ 下一篇已准备就绪                 ← 预生成完成

📌 发布第 2/3 篇: 周末居家办公小技巧
⚡ 使用预生成内容                    ← 直接使用
🔄 后台生成: 简单又好吃的快手菜

📌 发布第 3/3 篇: 简单又好吃的快手菜
⚡ 使用预生成内容

📊 发布完成: 成功 3/3 篇
```

**预期耗时**（3篇）：
- 标准模式: (1分钟生成 + 5分钟间隔) × 3 = 18分钟
- **优化模式: 1分钟首篇 + 5分钟×2 = 11分钟** ⚡
- **节省: 7分钟**

#### 测试3.3: 批量Dry-Run（标准模式对比）

```bash
python3 main.py batch --file test_topics.txt --dry-run --no-optimization
```

**对比观察**：
- 应该看到每篇都是"现场生成"，没有"使用预生成内容"的提示
- 耗时应该更长（约18分钟 vs 11分钟）

## 🐛 常见问题排查

### 问题1: API Key错误

**症状**：
```
❌ 文案生成失败: Invalid API key
```

**解决**：
```bash
# 检查配置
cat .env | grep OPENAI_API_KEY

# 确保没有多余空格
vim .env
```

### 问题2: Cookie失效

**症状**：
```
❌ Cookie 已失效，请重新登录
```

**解决**：
```bash
# 重新登录
python3 main.py login
```

### 问题3: 图片生成失败

**症状**：
```
❌ 图片生成失败: Rate limit exceeded
```

**解决**：
1. 等待几分钟再试
2. 检查API配额
3. 或使用本地图片：
```bash
python3 main.py publish --topic "测试" --images ./my_photos/
```

### 问题4: 浏览器启动失败

**症状**：
```
❌ Playwright error: Executable doesn't exist
```

**解决**：
```bash
# 重新安装浏览器
playwright install chromium
```
